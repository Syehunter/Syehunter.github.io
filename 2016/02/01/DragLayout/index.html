DragLayout--可拖拽的RecyclerView(类网易新闻频道管理实现)

发表于 2016-02-01   |  

原创博客，转载请注明出处： http://z.sye.space/2016/02/01/DragLayout/

之前公司的一个项目要求做一个新闻客户端，需要实现一个和网易新闻频道管理功能类似的界面。当时有用GridView去写，不过库写了一大半存在一点bug，新的需求又比较紧，就扔在一边没管了。

最近突然想起来，然后在想从前用GridView去实现，然而我们现在有了功能更加强大的RecyclerView，为什么不用RecyclerView来完成这样一个功能呢？

例行公事，先上效果图：

响应正常的click事件，长按模式下可进行拖动，点击item右上角的叉叉进行item的删除。删除的item会自动加到下面的一个RecyclerView中，并按照拼音顺序排序。

与传统的View.onDragListener相比，RecyclerView提供了新的api来处理拖动事件，也就是ItemTouchHelper。这个类实际上是ItemDecoration的子类，并且可以发现它提供了一个attachToRecyclerView(recyclerView)的方法，接收一个RecyclerView作为参数。

我们先看下ItemTouchHelper的构造方法：

 /**
 * Creates an ItemTouchHelper that will work with the given Callback.
 *
 * You can attach ItemTouchHelper to a RecyclerView via
 * {@link #attachToRecyclerView(RecyclerView)}. Upon attaching, it will add an item decoration,
 * an onItemTouchListener and a Child attach / detach listener to the RecyclerView.
 *
 * @param callback The Callback which controls the behavior of this touch helper.
 */
public ItemTouchHelper(Callback callback) {
    mCallback = callback;
}

从构造方法中可以看出创建ItemTouchHelper对象需要一个ItemTouchHelpder.Callback作为参数，从注释中解释了该Callback的作用：控制ItemTouchHelper的行为。也就是说这个ItemTouchHelper.Callback是控制item动作的主要部分。

ItemTouchHelper.Callback是一个公有的抽象静态类，于是定义一个DragItemTouchHelperCallback继承这个类，有三个方法需要我们自己去实现：

@Override
public int getMovementFlags(RecyclerView recyclerView, RecyclerView.ViewHolder viewHolder) {
    return 0;
}

@Override
public boolean onMove(RecyclerView recyclerView, RecyclerView.ViewHolder source, RecyclerView.ViewHolder target) {
        return false;
    }
    mOnItemChangeListener.onItemMove(source.getAdapterPosition(), target.getAdapterPosition());
    return true;
}

@Override
public void onSwiped(RecyclerView.ViewHolder viewHolder, int direction) {

}

实际上就是让使用者自己去完成对Move以及Swip时间的监听。getMovementFlags()这个方法比较简单，就是用来指定item支持的拖动或滑动方向，一般使用

  • ItemTouchHelper.UP
  • ItemTouchHelper.DOWN
  • ItemTouchHelper.START
  • ItemTouchHelper.END
  • ItemTouchHelper.LEFT
  • ItemTouchHelper.RIGHT

这几个参数来定义支持的方向，具体想要支持哪种自行选择，之后调用makeMovementFlags(dragFlags, swipFlags)来完成设置(注： dragFlags为拖动，swipFlags为滑动)。

onMove()和onSwiped()方法就需要我们自定去定义希望在触发移动和滑动时希望完成的事件，我这里定义了一个OnItemChangeListener<T>接口，其中<T>为RecyclerView.Adapter接收的List的数据类型。

public interface OnItemChangeListener<T> {

    boolean onItemMove(int fromPosition, int toPosition);

    void onItemRemoved(int position);

    void onItemInsert(int position, T data);
}

为DragItemTouchHelperCallback定义一个构造方法，接收OnItemChangeListener参数，于是我们就可以在刚刚提到的两个方法中做如下实现：

@Override
public boolean onMove(RecyclerView recyclerView, RecyclerView.ViewHolder source, RecyclerView.ViewHolder target) {
    if (source.getItemViewType() != target.getItemViewType()) {
        //the viewTypes between the source and the target are not same
        //can't move
        return false;
    }
            mOnItemChangeListener.onItemMove(source.getAdapterPosition(), target.getAdapterPosition());
    return true;
}

@Override
public void onSwiped(RecyclerView.ViewHolder viewHolder, int direction) {
    mOnItemChangeListener.onItemRemoved(viewHolder.getAdapterPosition());
}

之后只要让我们定义的Adapter去实现这个接口即可。

除此之外，我们还需要一些其他方法的支持：

@Override
public boolean isLongPressDragEnabled() {
    return true;
}

@Override
public boolean isItemViewSwipeEnabled() {
    return true;
}

第一个是方法是是否支持长按拖拽的动作，第二个是是否允许ItemView在被触摸后启用其滑动操作。

当开始拖动时，我们可能会希望改变item的背景、文字、图片巴拉巴拉来告诉用户这个view现在处于被拖动的状态，那么通过onSelectedChanged() clearView()两个方法控制ViewHolder的变化，比较简单直接上代码：

@Override
public void onSelectedChanged(RecyclerView.ViewHolder viewHolder, int actionState) {
    if (actionState != ItemTouchHelper.ACTION_STATE_IDLE) {
        //itemView is being dragged
        if (viewHolder instanceof BaseDragViewHolder) {
            ((BaseDragViewHolder) viewHolder).onDrag();
        }
    }
    super.onSelectedChanged(viewHolder, actionState);
}

@Override
public void clearView(RecyclerView recyclerView, RecyclerView.ViewHolder viewHolder) {
    super.clearView(recyclerView, viewHolder);

    viewHolder.itemView.setAlpha(1.0f);
    if (viewHolder instanceof BaseDragViewHolder) {
        ((BaseDragViewHolder) viewHolder).onDragFinished();
    }
}

在定义的BaseDragViewHolder中提供onDrag()及onDragFinished()两个public方法，具体想要如何变化可由子类自行实现。

定义我们的BaseDragAdapter<T, VH extends BaseDragViewHolder>，泛型T为Adapter要展示的List的数据类型，并实现OnItemChangeListener接口。

@Override
public boolean onItemMove(int fromPosition, int toPosition) {
    moveDataInList(fromPosition, toPosition);
    notifyItemMoved(fromPosition, toPosition);
    return true;
}

private void moveDataInList(int fromPosition, int toPosition) {
    if (fromPosition == toPosition) {
        return;
    }
    T item = mDatas.get(fromPosition);
    mDatas.remove(item);
    mDatas.add(toPosition, item);
}

@Override
public void onItemInsert(int position, T data) {
    //auto put the new item at the end
    mDatas.add(data);
    notifyItemInserted(mDatas.size() - 1);
    notifyItemRangeChanged(mDatas.size() - 1, mDatas.size());
}

分别做Item的移动、删除、插入的操作，并通知Adapter更新。在onBindViewHolder中对手指的长按和点击事件做了处理，常规方案，有希望了解的可以查看源码。值得一提的是在长按后调用了DragItemStartListener的onDragStart()方法，该listener由Adapter的构造方法传入，在自定义的RecyclerView中实现：

@Override
public void onDragStart(ViewHolder viewHolder) {
    mItemTouchHelper.startDrag(viewHolder);
}

startDrag()为ItemTouchHelper中的方法，通知ViewHolder开始拖拽。

我们还需要使RecyclerView能够自适应高度，RecyclerView的高度和传统的自定义View不同，是由LayoutManager控制的，Google一下就会有很多关于自定义高度的介绍，很多是LinearLayoutManager的，GridLayout实现的原理类似，重写onMeasure()方法，只不过GridLayout中每个item不再只占一行，所以需要重新计算，具体可看源码(注：并不支持设置了SpanSizeLookup的情况，如果需要在使用SpanSizeLookup的情况下适应高度，需要另行计算)。

核心的代码就是这些，其余的时一些关于Adapter的设置，对item的长按、点击、插入、删除的回调等，可查看源码自行理解。
